پیکربندی پایگاه داده¶

کدایگنایتر دارای فایل پیکربندی برای پایگاه داده است که به شما اجازه میدهد تا اطلاعات پایگاه داده خود را نگهداری کنید.(نام کاربری,پسورد,نام دیتابیس, ...). فایل پیکربندی در application/config/database.php واقع شده است. همچنین می‌توانید مقادیر پیکربندی پایگاه‌داده را برای محیط‌های خاص با قرار دادن database.php در پوشه کانفیگ محیط مربوطه ,تنظیم کنید .

تنظیمات پیکربندی پایگاه داده در یک آرایه چند بعدی, با این حالت پیشفرض ذخیره می شود:

$db['default'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'encrypt' => FALSE,
        'compress' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

برخی از مدل ها یا driver های اتصال پایگاه داده (مانند PDO، PostgreSQL، اوراکل، ODBC) ممکن است نیاز به DSN برای اجرا شدن و ارائه خدمات داشته باشند. در این صورت, شما باید از تنظیمات پیکر بندی “DSN” استفاده کنید, و اگر از الگو یا مدل PHP خالص برای اتصال استفاده میکنید به صورت زیر “DSN” را معرفی کنید:

// PDO
$db['default']['dsn'] = 'pgsql:host=localhost;port=5432;dbname=database_name';

// Oracle
$db['default']['dsn'] = '//localhost/XE';

نکته

اگر شما “DSN” را برای درایوری که نیاز به DSN دارد مشخص نکنید, کدایگنایتر با بقیه تنظیمات مشخص شده , سعی میکند که “DSN” را بسازد.

همچنین می توانید زمانی که اتصال پایگاه داده به هر دلیلی با شکست مواجه شد , failvoer یا حالت شکست مشخص کنید. در این حالت زمانی که آرایه پیشفرض نتواند متصل شود,آرایه دوم یا همان failover متصل خواهد شد. برای تنظیم failover میتوان به این شکل عمل نمود:

$db['default']['failover'] = array(
                array(
                        'hostname' => 'localhost1',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                ),
                array(
                        'hostname' => 'localhost2',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                )
        );

شما هر تعداد حالت شکست که میخواهید, میتوانید تعریف کنید.

دلیل استفاده از آرایه چند بعدی به جای آرایه یک بعدی, این است که به شما اجازه می‌دهد چندین حالت از مقادیر را برای اتصال به پایگاه داده ذخیره و استفاده کنید. به عنوان مثال, اگر شما چندین قسمت(توسعه, تولید , آزمایش و ...) , را همگی با هم اجرا کنید, میتوانید برای هر قسمت یک اتصال پایگاه داده جدا ایجاد کنید و در صورت نیاز از روی اتصال فعلی به روی اتصال مورد نیاز سوئیچ کنید. برای مثال، برای راه‌اندازی محیط "آزمایش" می‌توانید به صورت زیر عمل کنید:

$db['test'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'compress' => FALSE,
        'encrypt' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

سپس به سیستم بگویید, به صورت Global از این گروه استفاده کند، برای این کار می توانید این متغیر را که در فایل config قرار دارد تنظیم کنید:

$active_group = 'test';

نکته

نامی که شما برای آرایه انتخاب میکنید اختیاری است مانند "آزمایش". شما هر نامی که دوست دارید میتوانید برای آرایه قرار دهید. به صورت پیش فرض ما از واژه "default" استفاده کردیم, اما شما میتوانید نام آن را بسته به کار و پروژه خود تغییر دهید.

Query Builder¶

کلاس Query Builder با استفاده از متغیر query_builder$ در فایل پیکربندی پایگاه داده, به وسیله TRUE یا FALSE ,فعال یا غیرفعال می شود. مقدار پیشفرض این متغییر TRUE است. اگر از کلاس Query Builder استفاده نمیکنید , مقدار متغییر را به FALSE تغییر دهید , زیرا در هنگام راه اندازی پایگاه داده , منابع کمتری مصرف خواهد شد.

$query_builder = TRUE;

نکته

برخی از کلاس های Codeigniter مانند Session ها یا نشست ها به کوئری بیلدر ها نیاز دارند تا بتوانند به قابلیت ها و ویژگی ها دسترسی پیدا کنند.

معرفی قسمت های پیکربندی:¶

نام توضیح
dsn رشته اتصال DSN.
hostname نام میزبان سرور پایگاه داده در این قسمت مشخص میشود. پیشفرض ‘localhost’ است.
username نام کاربری مورد استفاده برای اتصال به پایگاه داده.
password پسورد مورد استفاده برای اتصال به پایگاه داده.
database نام پایگاه داده که قصد اتصال به آن را دارید.
dbdriver نوع پایگاه داده. مانند: odbc, postgre, mysqli و ... باید تمامی حروف کوچک باشند.
dbprefix یک پیشوند اختیاری به نام جدولی که در حال اجرای کوئری های Query Builder هست اضافه میکند.
pconnect TRUE/FALSE (boolean) - مشخص میکند که آیا میخواهید از اتصال مداوم استفاده کنید یا خیر.
db_debug TRUE/FALSE (boolean) - مشخص میکند که آیا خطا های پایگاه داده نمایش داده شود یا خیر.
cache_on TRUE/FALSE (boolean) - مشخص میکند که آیا query ها ذخیره شوند یا خیر, این قسمت را مطالعه کنید ذخیره سازی کوئری ها.
cachedir مسیر سرور به پوشه ذخیره سازی query ها را مشخص میکند.
char_set مشخص میکند که برای برقراری ارتباط به پایگاه داده از چه مدل encoding برای اطلاعات باید استفاده شود.
dbcollat

مشخص میکند از چه مدل کاراکترهایی برای برقراری ارتباط با پایگاه داده استفاده شود.

نکته

فقط در مدل های "My_sqli" و "My_sql" مورد استفاده قرار میگیرد.
swap_pre یک پیشوند پیشفرض جدول که با dbprefix عوض میشود.
schema schema پایگاه داده را مشخص میکند, به صورت پیشفرض ‘public’ است. برای مدل های PostgreSQL و ODBC مورد استفاده قرار میگیرد.
encrypt

مشخص میکند که از رمزنگاری برای اتصال استفاده شود یا خیر.

  • مدل های ‘mysql’ (منسوخ شده) , ‘sqlsrv’ و ‘PDO/sqlsrv’ مقدار را به صورت TRUE و FALSE قبول میکنند.
  • مدل های ‘mysqli’ و ‘PDO/mysql’ مقدار را به صورت یک آرایه قبول میکند که دارای گزینه های زیر است:
    • ‘ssl_key’ - مسیر فایل کلید خصوصی
    • ‘ssl_cert’ - مسیر فایل کلید عمومی
    • ‘ssl_ca’ - مسیر مرجع گواهی یا جواز کلید ها
    • ‘ssl_capath’ - مسیر دایرکتوری حاوی جواز معتبر CA در قالب PEM
    • ‘ssl_cipher’ - لیست Cipher یا رمز های مجازی که قرار است در رمز نگاری استفاده شود, و توسط کالن (‘:’) از هم جدا میشوند
    • ’TRUE/FALSE - ‘ssl_verify ; مشخص میکند که آیا گواهی یا جواز سرور تایید شود یا خیر,(فقط برای My_Sqli)
compress مشخص میکند که از فشرده سازی کلاینت استفاده شود یا خیر, (فقط برای My_sql).
stricton TRUE/FALSE (boolean) - مشخص میکند که آیا از حالت " Strict Mode " میخواهید استفاده کنید یا خیر.
port

پورت پایگاه داده را مشخص میکند. شما برای استفاده از این قسمت باید یک خط به آرایه پیکربندی اتصال اضافه کنید مانند زیر:

$db['default']['port'] = 5432;

نکته

شما بسته به پلتفرم پایگاه داده ایی که از آن استفاده میکنید (MySQL، PostgreSQL و غیره) نیازی نیست که از تمام مقادیر که در بالا گفته شد استفاده بکنید. به عنوان مثال , زمانی که شما از SQLite استفاده میکنید ,دیگر نیازی به وارد کردن نام کاربری و پسورد نخواهید داشت و همچنین نام دیتابیس شما, همان مسیر فایل دیتابیس شما خواهد بود. اطلاعات ذکر شده بالا فرض میکند که شما از MySQL استفاده میکنید.
